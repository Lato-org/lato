  <h1>Azioni</h1>

  
</div>

<p>
Le azioni permettono di visualizzare informazioni ed elementi funzionali in-page senza dover richiedere un cambio di pagina da parte dell'utente.<br>
Le azioni sono gestite tramite il controller di Stimulus JS <code>lato_action_controller</code>.
</p>

<p>
Per avviare un azione basta impostare l'attributo <code>data-lato-action-target="trigger"</code> ad un link del DOM e definire tramite l'attributo <code>data-turbo-frame="id_frame"</code> l'ID del turbo-frame da caricare nella azione. 
</p>

<p>
<a data-lato-action-target="trigger" data-turbo-frame="lato_index_default_product" href="products.html">Esempio</a>
</p>

<pre>
&lt;%= link_to 'Esempio',
  main_app.products_path,
  data: {
    lato_action_target: 'trigger',
    turbo_frame: 'lato_index_default_product'
  }
%&gt;
</pre>

<h3>Personalizzare l'overlay</h3>

<h4>Specificare il titolo</h4>

<p>
<a data-lato-action-target="trigger" data-turbo-frame="lato_index_default_product" data-action-title="Titolo personalizzato" href="products.html">Esempio</a>
</p>

<pre>
&lt;%= link_to 'Esempio',
  main_app.products_path,
  data: {
    lato_action_target: 'trigger',
    turbo_frame: 'lato_index_default_product',
    action_title: 'Titolo personalizzato'
  }
%&gt;
</pre>

<h4>Specificare la dimensione</h4>

<p>
<a data-lato-action-target="trigger" data-turbo-frame="lato_index_default_product" data-action-size="xl" href="products.html">Esempio</a>
</p>

<pre>
&lt;%= link_to 'Esempio',
  main_app.products_path,
  data: {
    lato_action_target: 'trigger',
    turbo_frame: 'lato_index_default_product',
    action_size: 'xl'
  }
%&gt;
</pre>
  </div>
</div>

  <h1>Componenti</h1>

  
</div>


<p>
Lato dispone di una serie di componenti funzionali riutilizzabili.<br>
Utilizzando le componenti di Lato è possibile risparmiare tempo di lavoro e facilitare le attività di manutenzione dell'applicazione.<br>
Tutte le componenti sono disponibili tramite l'helper <code>Lato::ComponentsHelper</code>.
</p>

<p>
<a target="_blank" href="https://github.com/lato-org/lato/blob/main/app/helpers/lato/components_helper.rb">Source code: Lato::ComponentsHelper</a>
</p>

<h2>Navbar</h2>

<h3>Navbar nav item</h3>

<p>
Componente per renderizzare una voce di menu nella navbar.
Parametri:<br>
- <i>key</i>: chiave della voce di menu. Utilizzata per attivare la voce di menu se impostata attiva tramite la funzione <code>active_navbar</code> del concern <code>Lato::Layoutable</code>.<br>
- <i>path</i>: path o url assoluto dove rimandare l'utente.<br>
- <i>&amp;block</i>: blocco di HTML da renderizzare all'interno della voce di menu.
</p>

<pre>
&lt;%= lato_navbar_nav_item :account, lato.account_path do %&gt;
  Account
&lt;% end %&gt;
</pre>

<h2>Sidebar</h2>

<h3>Sidebar nav item</h3>

<p>
Componente per renderizzare una voce di menu nella sidebar.
Parametri:<br>
- <i>key</i>: chiave della voce di menu. Utilizzata per attivare la voce di menu se impostata attiva tramite la funzione <code>active_sidebar</code> del concern <code>Lato::Layoutable</code>.<br>
- <i>path</i>: path o url assoluto dove rimandare l'utente.<br>
- <i>&amp;block</i>: blocco di HTML da renderizzare all'interno della voce di menu.
</p>

<pre>
&lt;%= lato_sidebar_nav_item :account, lato.account_path do %&gt;
  Account
&lt;% end %&gt;
</pre>

<h2>Form</h2>

<p>
I form possono essere generati utilizzando le componenti <code>lato_form</code>.<br>
Le componenti <code>lato_form</code> sono una serie di componenti diverse programmate per essere utilizzate con l'helper Rails <code>form_with</code> e il controller Stimulus <code>lato_form_controller</code>.<br>
Di seguito la lista di componenti disponibili e un esempio di utilizzo:
</p>
<ul>
  <li>
<code>lato_form_notices</code> visualizza il feedback positivo al salvataggio (notice impostati dalla action).</li>
  <li>
<code>lato_form_errors</code> visualizza il feedback negativo al salvataggio (errori presenti nell'istanza del modello).</li>
  <li>
<code>lato_form_item_label</code> visualizza la label dei campi.</li>
  <li>
<code>lato_form_item_input_text</code> visualizza un input di testo.</li>
  <li>
<code>lato_form_item_input_email</code> visualizza un input email.</li>
  <li>
<code>lato_form_item_input_password</code> visualizza un input password.</li>
  <li>
<code>lato_form_item_input_check</code> visualizza un input checkbox.</li>
  <li>
<code>lato_form_item_input_select</code> visualizza un input select.</li>
  <li>
<code>lato_form_submit</code> visualizza il pulsante di submit.</li>
</ul>

<h4>Personalizzare il contenuto delle colonne</h4>

<p>
Il contenuto delle colonne può essere personalizzato creando degli appositi helper con la sintassi <code>model_column(member)</code>.<br>
Per esempio, per personalizzare l'output della colonna :lato_user_id è stato creato il seguente helper in <code>ProductsHelper</code>:
</p>

<pre>
class ProductsHelper
  def product_lato_user_id(product)
    product.lato_user.full_name
  end
end
</pre>

<h4>Aggiungere colonne custom</h4>

<p>
Per aggiungere colonne custom ad una tabella basta specificare il nome del metodo dichiarato nel modello per fornire il dato corrispondente.<br>
Per esempio, dopo aver aggiunto il metodo <code>lifetime</code> al modello <code>Product</code>, è possibile utilizzare la colonna <code>:lifetime</code>:
</p>

<pre>
class Product &lt; ApplicationRecord
  # Helpers
  ##

  def lifetime
    Time.now - created_at
  end
end
</pre>

<pre>
class ProductsHelper
  def product_lifetime(product)
    Time.at(product.lifetime).utc.strftime('%H h %M m')
  end
end
</pre>

<div class="alert alert-info mt-3">
  <h4 class="alert-heading">Note</h4>
  <p>
  Per personalizzare le regole di ordinamento è possibile definire lo scope <code>lato_index_order</code> all'interno del modello.<br>
  Esempio
  </p>
<pre>
class Product &lt; ApplicationRecord
  # Relations
  ##

  belongs_to :lato_user, class_name: 'Lato::User'

  # Scopes
  ##

  scope :lato_index_order, -&gt;(column, order) do
    return joins(:lato_user).order("lato_users.last_name #{order}, lato_users.first_name #{order}") if column == :lato_user_id

    order("#{column} #{order}")
  end
end
</pre>

  Esempio
  </p>
<pre>
class Product &lt; ApplicationRecord
  # Relations
  ##

  belongs_to :lato_user, class_name: 'Lato::User'

  # Scopes
  ##

  scope :lato_index_search, -&gt;(search) do
    joins(:lato_user).where("
      lower(code) LIKE :search OR
      lower(lato_users.first_name) LIKE :search OR
      lower(lato_users.last_name) LIKE :search
    ", search: "%#{search.downcase.strip}%")
  end
end

  <h1>Configurazione</h1>

  
</div>

<p>
La configurazione di Lato è gestita dalla classe <code>Lato::Config</code> e può essere modificata tramite un apposito initializer.<br>
Per personalizzare la configurazione di default puoi creare un initializer è inserire il codice seguente:
</p>

<pre>
Lato.configure do |config|
  config.application_title = 'Lato example app'
  config.session_root_path = :tutorial_path
end
</pre>

<p>
<a target="_blank" href="https://github.com/lato-org/lato/blob/main/test/dummy/config/initializers/lato_config.rb">Source code: Esempio</a><br>
<a target="_blank" href="https://github.com/lato-org/lato/blob/main/lib/lato/config.rb">Source code: Lato::Config</a>
</p>

<h2>Configurazione applicazione</h2>

<pre>
Lato.configure do |config|
  config.application_title = 'Lato example app' # Titolo applicazione
  config.application_company_name = 'Lato Team' # Titolo azienda proprietaria dell'applicazione
  config.application_company_url = 'http://domain.com' # Url azienda proprietaria dell'applicazione
end
</pre>

<h2>Configurazione autenticazione</h2>

<pre>
Lato.configure do |config|
  config.auth_disable_signup = true # Disabilita la possibilità di registrarsi autonomamente
  config.auth_disable_recover_password = true # Disabilita la possibilità di recuperare la password autonomamente
  config.auth_disable_web3 = true # Disabilita la possibilità di autenticarsi utilizzando un wallet web3
  config.auth_disable_authenticator = true # Disabilita la possibilità di autenticarsi utilizzando google authenticator
end
</pre>

<h2>Configurazione assets</h2>

<h2>Configurazione informazioni legali</h2>

  config.legal_terms_and_conditions_url = 'http:/domain.com/terms' # Url termini e condizioni applicazione

<div class="alert alert-info mt-3">

user = Lato::User.all.first

</pre>

<h2>Configurazione classi layout</h2>

<p>

Gli elementi grafici che compongono il layout sono: <b>navbar</b>, <b>content</b>, <b>sidebar</b> e <b>footer</b>.<br>
Per personalizzare le classi di default puoi creare un initializer è inserire il codice seguente:
</p>

<pre>
Lato.bootstrap do |btstrap|
  btstrap.navbar = 'navbar-light navbar-expand-lg fixed-top bg-light shadow-sm px-3'
  btstrap.navbar_container = 'container-fluid'
  btstrap.navbar_collapse = 'justify-content-end'

  btstrap.sidebar = 'sidebar'

  btstrap.content = 'p-3'
  btstrap.content_container = 'container-fluid'

  btstrap.footer = 'bg-light px-3 py-2'
  btstrap.footer_container = 'container-fluid'
end
</pre>

<h2>Personalizzazione comportamento layout</h2>

<p>
Il comportamento del layout di Lato può essere personalizzato tramite le funzioni messe a disposizione dal concern <code>Lato::Layoutable</code>.<br>
Il concern disponde di una serie di funzioni utili. Di seguito un esempio:
</p>

    active_navbar(:tutorial) # imposta l'active sulla voce della navbar con chiave :tutorial

    page_class('custom-class') # aggiunge una classe nel body della pagina
  end
end
</pre>

  <h1>Operazioni</h1>

  
</div>

<p>
Le operazioni sono un sistema di Lato per gestire visualmente l'esecuzione di Job in background.<br>
Le operazioni sono gestite da diverse parti:
</p>

<ul>
  <li>Il modello <code>Lato::Operation</code> inizializza tutte le operazioni e tiene traccia del loro stato di esecuzione.</li>
  <li>Il controller <code>Lato::OperationsController</code> gestisce l'accesso degli utenti ad una operazione in corso.</li>
  <li>La componente <code>lato_operation</code> permette di visualizzare a front-end lo stato di una operazione.</li>
</ul>

<h3>Convertire un Job in una operazione</h3>

<p>
Il metodo più facile per convertire un Job in una operazione è, semplicemente, fare in modo che il Job sia una sotto-classe di <code>Lato::ApplicationJob</code>.<br>
È inoltre necessario che il metodo <code>perform()</code> riceva un solo parametro Hash <code>params</code>.
</p>

<pre>
# Prima
def MyJob &lt; ApplicationJob
  def perform(hello)
    puts hello
  end
end

# Dopo
def MyJob &lt; Lato::ApplicationJob
  def perform(params = {})
    puts params['hello']
  end
end
</pre>

<h3>Generare una operazione</h3>

<p>
Le operazioni possono essere generate tramite il metodo <code>generate</code> del modello <code>Lato::Operation</code>.<br>
La generazione di un'operazione richiede tre parametri: nome del job da eseguire, Hash dei parametri da passare, id dell'utente che ha richiesto la generazione.
</p>

<pre>
operation = Lato::Operation.generate('ExportProductsJob', { product_ids: [1, 2, 3] }, @session.user_id)
</pre>

<p>
Una volta generata l'operazione deve essere eseguita attraverso il metodo <code>start</code>.<br>
Di seguito un esempio di controller che avvia una operazione:
</p>

<pre>
class CustomController &lt; ApplicationController
  def create_operation_action
    @operation = Lato::Operation.generate('OperationExampleJob', {}, @session.user_id)

    respond_to do |format|
      if @operation.start
        format.html { redirect_to lato.operation_path(@operation) }
        format.json { render json: @operation }
      else
        format.html { render :index, status: :unprocessable_entity }
        format.json { render json: @operation.errors, status: :unprocessable_entity }
      end
    end
  end
end
</pre>

<h3>Mostrare l'esito di una operazione</h3>

<p>
Ogni operazione può essere visualizzata nella route <code>lato.operation_path(operation_id)</code>.<br>
Per garantire la sicurezza dei dati un'operazione è accessibile solo per l'utente che ne ha richiesto la generazione.
</p>

<p>
Ogni operazione può avere 4 stati: <code>created</code>, <code>running</code>, <code>completed</code>, <code>failed</code>.<br>
La visualizzazione di una operazione, gestita tramite la componente <code>lato_operation</code>, mostra in real-time lo stato in cui si trova l'operazione e il relativo output.<br>
Utilizza il link di seguito per provare una operazione:
</p>

<pre>
&lt;%= link_to 'Genera operazione',
  main_app.custom_create_operation_action_path,
  data: { turbo_method: :post }
%&gt;

<h2>Come proteggere una pagina</h2>

<p>
Per autenticare una pagina in modo da non renderla pubblica per utenti non registrati puoi utilizzare uno dei metodi messi a dispozione dal concern <code>Lato::Sessionable</code> (già importato da <code>Lato::ApplicationController</code>):
</p>

<pre>
class TutorialController &lt; ApplicationController
  def index
    return unless authenticate_session
  end
end
</pre>

<p>
Per applicare la regola su tutte le action del controller puoi utilizzare rapidamente il <code>before_action</code>:
</p>

<pre>
class TutorialController &lt; ApplicationController
  before_action :authenticate_session

  def index; end
end
</pre>

<div class="alert alert-info mt-3">
  <h4 class="alert-heading">Note</h4>
  <p>
  Proprio come <code>authenticate_session</code> puoi anche utilizzare il contrario <code>not_authenticate_session</code> per nascondere una pagina agli utenti registrati.<br>
  Esempio:
  </p>
<pre>
class AuthenticationController &lt; ApplicationController
  before_action :not_authenticate_session, only: %i[signin]
  before_action :authenticate_session, only: %i[signout]

  def signin; end

  def signout; end
end
</pre>
</div>

<h3>Limitare il numero di richieste</h3>

<p>
Per limitare il numero di richieste di una pagina puoi utilizzare il metodo <code>limit_request</code>:
</p>

<pre>
class TutorialController &lt; ApplicationController
  before_action :authenticate_session
  before_action :limit_request, only: %i[index]

  def index; end
end
</pre>

<p>
Il metodo <code>limit_request</code> accetta due parametri: <code>limit</code> (default: 10) e <code>time_window</code> (default: 10.minutes).<br>
I due parametri specificano il numero di richieste massime che un utente può effettuare in un determinato lasso di tempo.<br>
</p>

<p>
<a target="_blank" href="https://github.com/lato-org/lato/blob/main/app/controllers/concerns/lato/sessionable.rb">Source code Lato::Sessionable</a>
</p>

<h2>Accedere all'utente loggato</h2>

<p>
L'utente che ha effettuato l'accesso a Lato è sempre disponibile in tutti i controller e tutte le view tramite l'istanza <code>@session</code> del modello <code>Lato::Session</code>:
</p>

<pre>
class TutorialController &lt; ApplicationController
  # ...

  def index
    @user_id = @session.user_id
    @user = @session.user.first_name
  end

  # ...
end
</pre>

<div class="alert alert-info mt-3">
  <h4 class="alert-heading">Note</h4>
  <p>
  Utilizzando <code>@session.user_id</code> al posto di <code>@session.user.id</code> è possibile evitare di eseguire query nella tabella utenti ottimizzando i tempi di risposta!
  </p>
  <p>
  <code>@session.valid?</code> permette di verificare se l'utente è loggato o meno.
  </p>
</div>

<p>
<a target="_blank" href="https://github.com/lato-org/lato/blob/main/app/models/lato/session.rb">Source code Lato::Session</a>
</p>
